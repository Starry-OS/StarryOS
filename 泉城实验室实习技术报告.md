```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum TestError {
    ReadError,        // 块设备读错误
    WriteError,       // 块设备写错误
    DataMismatch,     // 数据校验失败
    InvalidBlock,     // 无效块地址
    Timeout,          // 设备超时
    AssertionFailed,  // 断言失败

    IoError,          // 通用I/O错误
}

/// 性能测试统计信息
#[derive(Debug, Default, Clone)]
pub struct TestStats {
    pub bytes_transferred: u64,  // 传输字节数
    pub elapsed_nanos: u64,      // 耗时（纳秒）
    pub operations: u64,         // 操作次数
    pub errors: u64,             // 错误次数
}

impl TestStats {
    /// 计算吞吐量 (MB/s)
    pub fn throughput_mbps(&self) -> f64 {
        if self.elapsed_nanos == 0 {
            return 0.0;
        }
        let bytes = self.bytes_transferred as f64;
        let seconds = self.elapsed_nanos as f64 / 1_000_000_000.0;
        bytes / seconds / (1024.0 * 1024.0)
    }

    /// 计算IOPS
    pub fn iops(&self) -> f64 {
        if self.elapsed_nanos == 0 {
            return 0.0;
        }
        let seconds = self.elapsed_nanos as f64 / 1_000_000_000.0;
        self.operations as f64 / seconds
    }
}
```

##### 将 `sdmmc-tests` 接入 Starry

按照以下步骤可以在 StarryOS 项目根目录中本地接入 `sdmmc-tests`：

1. 将 `sdmmc-tests` 加入 Starry 的 Workspace 中。

    ``` toml
    # StarryOS/Cargo.toml

    [workspace]
    resolver = "2"
    members = ["kernel", "sdmmc-tests"]
    exclude = ["arceos"]
    ```

2. 将 `sdmmc-tests` 作为 `kernel` crate 的依赖。

    ``` toml
    # StarryOS/kernel/Cargo.toml

    [dependencies]
    sdmmc_tests = { path = "../sdmmc-tests", default-features = false }
    ```

3. 在 `kernel` crate 的 `init` 函数中调用 `sdmmc-tests` 的 `run_all_tests` 函数运行所有测试。

    ``` rust
    //  StarryOS/kernel/src/entry.rs

    pub fn init(args: &[String], envs: &[String]) {
        pseudofs::mount_all().expect("Failed to mount pseudofs");
        spawn_alarm_task();

        if let Err(e) = run_all_tests() {
            panic!("SD/MMC tests failed during kernel init: {:?}", e);
        }

        ...
    }
    ```

### 为现有的 SD 卡驱动（基于轮询）增加了 DMA 读写功能

修改后的代码放在[simple-sdmmc-extended GitHub 仓库](https://github.com/PassingThroughStarDust/simple-sdmmc-extended)中。

#### 驱动 DMA 功能介绍

1. IDMAC 初始化流程

   1. **分配并初始化描述符环（Descriptor Ring）**
      - 分配一块物理连续的内存作为 DMA 描述符环（`sg_cpu`/`sg_dma`），并初始化所有描述符的链表关系。
      - 设置每个描述符的下一个描述符地址，最后一个描述符设置为环形（End-of-Ring）。

   2. **IDMAC 控制器复位**
      - 通过写 BMOD（Bus Mode Register）和 CTRL（Control Register）相关位，软复位 DMA 控制器。

   3. **配置中断**
      - 屏蔽所有中断，仅使能必要的 DMA 传输完成（NI, RI, TI）中断。

   4. **设置描述符基地址**
      - 将描述符环的物理首地址写入 DBADDR（Descriptor List Base Address Register）。

2. DMA 方式读写 SD 卡流程

   1. **准备数据传输参数**
      - 设置块大小、传输字节数等参数。
      - 填写描述符内容，包括数据缓冲区物理地址、长度、控制位（OWN、CH、FD、LD 等）。

   2. **启动 DMA 传输**
      - 选择 IDMAC 接口（CTRL 寄存器 USE_INTERNAL_DMAC 位置 1）。
      - 使能 IDMAC（BMOD 寄存器 DE 位置 1）。
      - 向 PLDMND（Poll Demand Register）写任意值，触发 FSM 启动。

   3. **等待传输完成**
      - 通过中断或轮询方式等待 DMA 传输完成。
      - 传输完成后，驱动会调用 `mci_dmac_complete_dma`，进行缓存同步、状态清理等。

   4. **停止/复位 DMA**
      - 传输完成或异常时，可调用 `idmac_stop_dma` 停止 DMA，并通过 `idmac_reset_dma` 进行软复位。
      - 清理描述符、状态等。

2. 基于 IDMAC 的 DMA 读写接口函数

    `SdMmc` 具有以下接口函数以启用 IDMAC 进行并进行数据块的读取：
    - **idmac_init_dma**
       - 初始化 IDMAC 控制器。
       - 设置并初始化 DMA 描述符环（分配、链表连接、End-of-Ring 标记）。
       - 配置 DMA 相关中断。
       - 写入描述符基地址。
       - 复位 DMA 控制器，确保状态干净。

    - **idmac_start_dma**
       - 启动 DMA 传输。
       - 选择 IDMAC 作为数据通道。
       - 使能 IDMAC。
       - 向 PLDMND 写入，触发 DMA 状态机开始传输。

    - **idmac_stop_dma**
       - 停止 DMA 传输。
       - 禁用 IDMAC 并复位相关控制器状态。
       - 适用于传输完成或异常终止场景。

    - **idmac_reset_dma**
       - 对 DMA 控制器进行软复位。
       - 清除所有状态，确保后续传输不会受前一次影响。

    - **idmac_cleanup_dma**
       - 清理 DMA 状态。
       - 通常在传输完成或异常后调用，确保 DMA 资源和状态恢复。

    - **mci_prepare_desc32**
       - 构建 32 位模式下的 DMA 描述符链表。
       - 填写每个描述符的控制位、数据缓冲区地址、长度等。
       - 支持 scatter-gather（多段）传输。

    - **mci_dmac_complete_dma**
       - DMA 传输完成后的回调处理。
       - 进行缓存同步（如有）、状态清理、通知上层传输完成。

    ``` rust
    impl SdMmc {
        // DMA functions for IDMAC (Internal DMA Controller) of DWC_MSHC
        pub fn idmac_init_dma(&mut self) {
            trace!("Initializing IDMAC for DMA transfer");

            // Setup descriptor ring
            // Only 32-bit mode supported for VisionFive2
            let ring_size = Self::DESC_RING_BUF_SZ / core::mem::size_of::<IdmacDescriptor>();
            self.dma_descriptor_ring_size = ring_size;

            // Forward link the descriptor list
            unsafe {
                let mut p = self.sg_cpu;
                for i in 0..ring_size - 1 {
                    (*p.add(i)).set_next_descriptor_address(self.sg_dma as u32 + ((i as u32 + 1) * core::mem::size_of::<IdmacDescriptor>() as u32));
                    (*p.add(i)).set_control_descriptor(Default::default());
                    (*p.add(i)).set_buffer1_size(0);
                    (*p.add(i)).set_buffer1_address(0);
                }
                // Last descriptor: end-of-ring
                (*p.add(ring_size - 1)).set_next_descriptor_address(self.sg_dma as u32);
                let mut des0 = Default::default();
                des0.set_er(true);
                (*p.add(ring_size - 1)).set_control_descriptor(des0);
            }

            self.idmac_reset_dma();

            // Mask out interrupts - get Tx & Rx complete only
            self.regs.idsts().write(0xFFFFFFFF); // Clear all
            self.regs.idinten().write(0x7); // Enable NI, RI, TI

            // Set the descriptor base address
            self.regs.dbaddr().write(self.sg_dma as u32);
        }

        pub fn idmac_reset_dma(&mut self) {
            trace!("Resetting IDMAC DMA controller and internal state");
            // Software reset of DMA
            self.regs.bmod().update(|r: crate::regs::BMod| r.with_swr(true));
        }

        pub fn idmac_start_dma(&mut self) {
            trace!("Starting IDMAC DMA transfer");
            // Select IDMAC interface
            self.regs.ctrl().update(|r: crate::regs::Ctrl| r.with_use_internal_dmac(true));
            // Enable the IDMAC
            self.regs.bmod().update(|r: crate::regs::BMod| r.with_de(true));
            // Start it running
            self.regs.pldmnd().write(1);
        }

        pub fn idmac_stop_dma(&mut self) {
            trace!("Stopping IDMAC DMA transfer");
            // Disable and reset the IDMAC interface
            self.regs.ctrl().update(|r: crate::regs::Ctrl| r.with_use_internal_dmac(false).with_dma_reset(true));
            // Stop the IDMAC running
            self.regs.bmod().update(|r: crate::regs::BMod| r.with_de(false).with_fb(false).with_swr(true));
        }

        pub fn idmac_cleanup_dma(&mut self) {
            trace!("Cleaning up IDMAC DMA state");
            // Clean up DMA state, usually by resetting the DMA controller
            self.idmac_reset_dma();
            // TODO: Unmap DMA if needed (not required for simple baremetal)
        }

        fn mci_prepare_desc32(&self) {
            // This would setup the descriptors for a scatter-gather list.
            // In this simplified version, assume a single buffer transfer.
            // The actual implementation would walk the scatter-gather list and fill descriptors.
            // For now, just set up the first descriptor for a single buffer.
            unsafe {
                let desc = &mut *self.sg_cpu;
                let mut des0 = Default::default();
                des0.set_own(true);
                des0.set_ch(true);
                des0.set_fd(true);
                des0.set_ld(true);
                desc.set_control_descriptor(des0);
                desc.set_buffer1_size(self.mmc_data_info.byte_cnt as u16);
                // desc.set_buffer1_address(...); // Set by caller
                // desc.set_next_descriptor_address(...); // Set by caller
            }
        }

        fn mci_dmac_complete_dma(&self) {
            // Called when DMA transfer is complete
            trace!("DMA transfer complete");
            // Invalidate cache if needed (for read)
            // In baremetal, this may be a no-op
            // Clean up DMA state
            self.idmac_cleanup_dma();
            // Signal transfer complete (could set a flag or call a callback)
        }
    }
    ```

3. IDMAC 描述符链数据结构

    ``` rust
    #[repr(C, align(4))]
    pub struct IdmacDescriptor {
        pub des0: IdmacDes0,
        pub des1: IdmacDes1,
        pub des2: u32,
        pub des3: u32,
    }

    /// IDMAC Descriptor Flags
    #[bitfield(u32, order = Msb)]
    pub struct IdmacDes0 {
        pub own: bool,
        pub ces: bool,

        #[bits(24)]
        pub _reserved1: u32,

        pub er: bool,
        pub ch: bool,
        pub fs: bool,
        pub ld: bool,
        pub dic: bool,

        // Reserved bit (bit 0)
        #[bits(1)]
        pub _reserved0: u8,
    }

    #[bitfield(u32, order = Msb)]
    pub struct IdmacDes1 {
        #[bits(19)]
        pub _reserved: u32,

        #[bits(13)]
        pub bs1: u16,
    }

    impl IdmacDescriptor {
        pub fn new() -> Self {
            trace!("Creating a new IDMAC Descriptor with default values");
            Self {
                des0: IdmacDes0::default(),
                des1: IdmacDes1::default(),
                des2: 0,
                des3: 0,
            }
        }

        /// Sets the control bits for the DMA transfer in des0.
        pub fn set_control_descriptor(&mut self, des0: IdmacDes0) {
            trace!(
                "Setting control descriptor:\nown={}, ces={}, er={}, ch={}, fs={}, ld={}, dic={}",
                des0.own(), des0.ces(), des0.er(), des0.ch(), des0.fs(), des0.ld(), des0.dic()
            );

            self.des0 = des0;
        }

        pub fn set_buffer1_size(&mut self, size: u16) {
            trace!("Setting buffer1 size: {}", size);

            // size.le()
            self.des1.set_bs1(size);
        }

        /// Sets the address of the first data buffer for the DMA transfer in des2.
        pub fn set_buffer1_address(&mut self, addr: u32) {
            trace!("Setting buffer1 address: 0x{:08x}", addr);

            self.des2 = addr;
        }

        pub fn set_next_descriptor_address(&mut self, addr: u32) {
            trace!("Setting next descriptor address: 0x{:08x}", addr);

            self.des3 = addr;
        }
    }
    ```
